  <name>Task 1: Migrate App.tsx to use getAudioEngine() singleton and getTrackFacade()</name>
  <files>src/App.tsx</files>
  <action>
**Understanding the current state:**

App.tsx currently:
1. Calls `useAudioEngine()` which returns `AudioEngine | null` (null while loading)
2. Has `AppWithEngine` component that receives the engine and uses APP_* constants to look up modules
3. Uses `buildUiRuntime()` to map the UI plan into runtime models
4. Hardcodes APP_TRACK_STRIP_ID comparisons for the "is this the primary track?" branching

**Migration strategy:**

The singleton engine is always available (no null state, no loading). The old `AudioEngine` interface with `getSynth(id)`, `getPanner(id)`, etc. still works for device access. The singleton provides track/master facades for strip-level access, plus `_legacy` accessors for bridging.

For this migration, the app must remain FUNCTIONALLY IDENTICAL. The singleton does NOT yet manage devices (synth, panner) — those remain accessed through the old AudioEngine pattern. The key change is: strip-level access goes through facades, and the useAudioEngine hook is eliminated.

**Step-by-step implementation:**

1. **Replace `useAudioEngine()` with direct singleton access:**
   ```typescript
   import { getAudioEngine, DEFAULT_TRACK_ID } from './engine/engineSingleton';
   ```
   Remove the `useAudioEngine` import entirely.

2. **Keep the old engine creation for device access.** The singleton's `_legacy.audioContext` provides the AudioContext. Import `createAudioEngine` and the plan/factory map directly, and create the old engine at module level using the SAME AudioContext (they share it via Tone.js):
   ```typescript
   import { createAudioEngine, DEFAULT_AUDIO_MODULE_FACTORY_MAP } from './engine/audioEngine';
   import { DEFAULT_AUDIO_GRAPH_PLAN, DEFAULT_PLAN_SYNTH_ID, DEFAULT_PLAN_PANNER_ID, DEFAULT_PLAN_LIMITER_ID } from './engine/audioGraphPlan';
   ```
   The old engine is used ONLY for device lookups (getSynth, getPanner, getLimiter). Track strip and master strip come from the singleton facades.

3. **Remove the null-engine loading guard** — the singleton is always available, so the `if (!audioEngine) return <div id="app" />` check and the `AppWithEngine` split are unnecessary. Flatten everything back into a single `App` component.

4. **Replace APP_TRACK_STRIP_ID usage in trackZoneModel mapping** — Instead of comparing `runtimeTrack.trackStripId === APP_TRACK_STRIP_ID`, use `getAudioEngine().getTrackFacade(DEFAULT_TRACK_ID)` to access track strip data. Since there's only one track right now, simplify the branching: all tracks use the facade uniformly.

5. **Replace APP_MASTER_STRIP_ID usage** — Use `getAudioEngine().getMasterFacade()` for master volume and meter. Create an adapter for the `MasterStripHook` interface expected by existing hooks:
   ```typescript
   const masterFacade = engine.getMasterFacade();
   ```

6. **Replace APP_SYNTH_MODULE_ID and APP_PANNER_MODULE_ID** — Replace with `DEFAULT_PLAN_SYNTH_ID` and `DEFAULT_PLAN_PANNER_ID` imported from `audioGraphPlan.ts`. These are the same string values; we're just removing the indirection through `findDeviceModuleIdByKindOrThrow`.

7. **Replace APP_LIMITER_MODULE_ID** — Replace with `DEFAULT_PLAN_LIMITER_ID` from `audioGraphPlan.ts`.

8. **Remove `findDeviceModuleIdByKindOrThrow` function** — no longer needed since we import the IDs directly.

9. **Remove `INITIAL_TRACK_PLAN` lookup** — replace with `DEFAULT_TRACK_ID` where the track ID is needed.

10. **Update `buildUiRuntime` call** — `buildUiRuntime.ts` takes an `AudioEngine` typed parameter. Pass the old engine object for device resolution. `buildUiRuntime.ts` is kept as-is with NO modifications — it continues to receive an `AudioEngine` and resolve devices through it. The old engine and the singleton share the same AudioContext (both use Tone.js's context), so all audio nodes are on the same graph.

11. **Update the devicePanelModel mapping** — replace `device.moduleId === APP_SYNTH_MODULE_ID` with `device.moduleId === DEFAULT_PLAN_SYNTH_ID`, etc.

12. **Update trackZoneActions** — replace the `runtimeTrack.trackStripId === APP_TRACK_STRIP_ID` branches. Since all tracks now go through facades, use facade methods uniformly. For the transport controller's setTrackMute, the trackStrip hook still works for the default track.

**Key wiring that must be preserved:**
- Synth output -> panner input -> panner output -> track strip input (the device chain into the track)
- Track strip output -> preLimiterBus (already wired by singleton)
- preLimiterBus -> limiter -> master strip -> destination (already wired by singleton)
- Transport controller receives synth and track strip hooks
- MIDI keyboard receives synth hook

**What to delete:**
- `import { useAudioEngine } from './hooks/useAudioEngine'`
- All five `APP_*` constant definitions
- `findDeviceModuleIdByKindOrThrow` function
- `INITIAL_TRACK_PLAN` lookup
- The `AppWithEngine` component split (flatten back to `App`)
- The null-engine loading guard

**What to keep/update:**
- All `use*` hook calls (useToneSynth, usePanner, useTrackStrip, useMasterStrip, useLimiter) — they receive adapted inputs
- `buildUiRuntime` call — passes the old engine for device resolution. **buildUiRuntime.ts itself is NOT modified** — it receives the same `AudioEngine` interface it always has
- All component rendering (Toolbar, TrackZone, DevicePanel, MidiKeyboard) — identical
