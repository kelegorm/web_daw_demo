Create `src/state/DawStore.ts` implementing the BLoC pattern:

**Class structure** — uses private class fields (following TrackFacadeImpl convention from Phase 1):

```
class DawStore {
  #state: DawState
  #engine: EngineApi (type-only — import the interface from engineSingleton.ts)
  #listeners: Set<() => void>
  #idService: ReturnType<typeof createIdService>
  #projectSnapshot: ProjectDocument
  #uiSnapshot: UiState
}
```

**Constructor:**
- Accepts `engine: EngineApi` and `initialState: DawState` parameters.
- Stores engine reference, initializes state, creates idService.
- Seeds idService with all existing track IDs from initialState (`initialState.project.tracks.ids`).
- Initializes `#projectSnapshot` and `#uiSnapshot` with `Object.freeze({ ...initialState.project })` and `Object.freeze({ ...initialState.ui })` respectively.

**Snapshot methods** — arrow-method class fields for stable reference identity (per RESEARCH.md Pattern 1):
- `getProjectSnapshot = (): ProjectDocument => this.#projectSnapshot` — returns cached reference, NEVER constructs new object.
- `getUiSnapshot = (): UiState => this.#uiSnapshot` — returns cached reference.
- `subscribe = (listener: () => void): (() => void)` — adds listener to Set, returns unsubscribe function that deletes from Set.

**Internal dispatch** — private method:
- `#dispatch(action: DawAction): void`
- Calls `dawReducer(this.#state, action)` to get new state.
- Only updates snapshots if state actually changed (reference equality check on project/ui slices).
- If project changed: `this.#projectSnapshot = Object.freeze({ ...newState.project })`
- If ui changed: `this.#uiSnapshot = Object.freeze({ ...newState.ui })`
- Updates `this.#state = newState`
- Calls `this.#notify()` — notifies all listeners.

**#notify():** — iterates `#listeners` and calls each.

**Public methods — engine first, state second:**

`addTrack(): string`
1. Generate new ID via `this.#idService.generate()`
2. Generate display name: `Track ${this.#state.project.tracks.ids.length + 1}` (simple incrementing name)
3. Call `this.#engine.createTrackSubgraph(id)` — ENGINE FIRST
4. Call `this.#dispatch({ type: 'ADD_TRACK', id, displayName })` — STATE SECOND
5. Return the new track ID

`removeTrack(id: string): void`
1. Guard: if `this.#state.project.tracks.ids.length <= 1`, return early (min-1 track)
2. Guard: if track ID not in `this.#state.project.tracks.byId`, return early
3. Call `this.#engine.removeTrackSubgraph(id)` — ENGINE FIRST
4. Call `this.#dispatch({ type: 'REMOVE_TRACK', id })` — STATE SECOND

`selectTrack(id: string): void`
1. Call `this.#dispatch({ type: 'SELECT_TRACK', id })` — UI only, no engine call

**Export:** Export `DawStore` class as named export. Also export the `EngineApi` type if needed (or re-export from engineSingleton).

**CRITICAL constraints:**
- Arrow-method class fields for subscribe, getProjectSnapshot, getUiSnapshot (stable references for useSyncExternalStore).
- `Object.freeze` on snapshots to prevent accidental mutation.
- Engine called BEFORE dispatch — if engine throws, state is never modified.
- No React imports whatsoever.

**Test cases:**

**Construction:**
- Creates with initial state and engine without throwing
- `getProjectSnapshot()` returns initial project data
- `getUiSnapshot()` returns initial ui state

**addTrack():**
- Calls `engine.createTrackSubgraph` with generated ID (engine first)
- After addTrack, `getProjectSnapshot().tracks.ids` contains new track ID
- After addTrack, `getUiSnapshot().selectedTrackId` is the new track ID (auto-select)
- Returns the new track ID
- Notifies listeners (subscribe callback called)
- Two successive addTrack calls produce different IDs

**removeTrack():**
- Calls `engine.removeTrackSubgraph` with the track ID (engine first)
- After removeTrack, track is gone from `getProjectSnapshot().tracks.byId`
- After removeTrack, track is gone from `getProjectSnapshot().tracks.ids`
- When removing selected track, `getUiSnapshot().selectedTrackId` changes to adjacent
- When only 1 track remains, removeTrack is a no-op (does not call engine, state unchanged)
- When track ID doesn't exist, removeTrack is a no-op

**selectTrack():**
- Updates `getUiSnapshot().selectedTrackId`
- Does NOT call any engine methods
- Notifies listeners

**Snapshot stability (CRITICAL — prevents infinite re-render):**
- `getProjectSnapshot()` returns same reference on successive calls when state unchanged
- `getUiSnapshot()` returns same reference on successive calls when state unchanged
- After addTrack, `getProjectSnapshot()` returns a NEW reference (different from before)
- After selectTrack, `getProjectSnapshot()` returns the SAME reference (project didn't change)
- After selectTrack, `getUiSnapshot()` returns a NEW reference

**subscribe/unsubscribe:**
- Subscribe returns an unsubscribe function
- After unsubscribe, listener is not called on state changes
- Multiple listeners all get notified

**Engine-first ordering:**
- If `engine.createTrackSubgraph` throws, state is unchanged (getProjectSnapshot same as before)
- If `engine.removeTrackSubgraph` throws, state is unchanged

**ID service integration:**
- After adding and removing tracks, new addTrack never reuses a previously used ID
