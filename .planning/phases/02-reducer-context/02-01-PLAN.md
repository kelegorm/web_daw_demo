**src/state/types.ts** — Pure type definitions only, no runtime code:
- `Track`: `{ readonly id: string; readonly displayName: string; readonly deviceIds: readonly string[]; readonly clipIds: readonly string[] }`
- `Device`: `{ readonly id: string; readonly kind: 'SYNTH' | 'PANNER' | 'LIMITER'; readonly displayName: string }`

- `NormalizedMap<T>`: `{ readonly byId: Readonly<Record<string, T>>; readonly ids: readonly string[] }`
- `ProjectDocument`: `{ readonly tracks: NormalizedMap<Track>; readonly devices: Readonly<Record<string, Device>>; readonly clips: Readonly<Record<string, MidiClip>>; readonly masterTrack: MasterTrack }`
- `UiState`: `{ readonly selectedTrackId: string }`
- `DawState`: `{ readonly project: ProjectDocument; readonly ui: UiState }` (combined type for DawStore internal state)
- Import and re-export `MidiClip` and `MidiStep` from `../../project-runtime/midiClipStore` so consumers import from `src/state/types` instead of reaching into project-runtime directly.

**src/state/actions.ts** — Discriminated union:
- `type DawAction = AddTrackAction | RemoveTrackAction | SelectTrackAction`
- `AddTrackAction`: `{ type: 'ADD_TRACK'; id: string; displayName: string }`
- `RemoveTrackAction`: `{ type: 'REMOVE_TRACK'; id: string }`
- `SelectTrackAction`: `{ type: 'SELECT_TRACK'; id: string }`
- Export the union type and each individual action type.

**src/state/idService.ts** — Encapsulated ID generation:
- Export `createIdService()` factory function returning `{ seed, generate }`.
- `seed(ids: string[]): void` — registers pre-existing IDs into the used set.
- `generate(): string` — returns a unique ID string that has never been returned before. Algorithm: incrementing counter prefixed with `track-` (e.g., `track-2`, `track-3`). Start counter at 1. If generated ID collides with a seeded ID, increment and retry.
- Use a `Set<string>` internally for O(1) lookup of used IDs.
- Keep it simple — incrementing counter is fine for a demo app (per CONTEXT.md "Claude's Discretion").

**src/state/idService.test.ts** — Unit tests (node environment, NO @vitest-environment comment):
- `generate()` returns unique IDs on successive calls
- `seed()` prevents seeded IDs from being generated
- After seeding `['track-1', 'track-2']`, `generate()` skips to `track-3`
- Generated IDs are never reused even after many calls
- `seed()` can be called multiple times (additive)

**src/state/projectReducer.ts** — Pure function `(ProjectDocument, DawAction) => ProjectDocument`:
- `ADD_TRACK`: Adds new track to `byId` and appends ID to `ids`. New track has empty `deviceIds: []` and `clipIds: []`.
- `REMOVE_TRACK`: Removes track from `byId`, filters from `ids`. Does NOT enforce min-1 (that's DawStore's job — reducer is pure data transform).
- `SELECT_TRACK`: Returns state unchanged (projectReducer doesn't handle selection).
- Default case: `const _exhaustive: never = action; return _exhaustive;` — exhaustive check.
- CRITICAL: No engine imports. No side effects. Pure function.

**src/state/uiReducer.ts** — Pure function `(UiState, DawAction, ProjectDocument) => UiState`:
- Note: uiReducer receives current ProjectDocument as third arg so it can compute adjacent track on removal.
- `ADD_TRACK`: Sets `selectedTrackId` to `action.id` (auto-select new track).
- `REMOVE_TRACK`: If `state.selectedTrackId === action.id`, compute adjacent track from the provided ProjectDocument's `tracks.ids` (prefer next, fall back to previous). If selectedTrackId is not the removed track, return state unchanged.
- `SELECT_TRACK`: Sets `selectedTrackId` to `action.id`.
- Default case: exhaustive never check.

**src/state/dawReducer.ts** — Delegates to both reducers:
- `dawReducer(state: DawState, action: DawAction): DawState`
- Calls `projectReducer(state.project, action)` first.
- Calls `uiReducer(state.ui, action, state.project)` second — note: passes the OLD project state so uiReducer can see the track list BEFORE removal (needed for adjacent track computation).
- Returns `{ project: newProject, ui: newUi }`.
- CRITICAL: No engine imports. Pure function.
