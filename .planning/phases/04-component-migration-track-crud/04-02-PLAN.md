**1b. Handle transport state:**

TrackZone needs playbackState, bpm, loop, and getPositionSeconds for the playhead/timeline. These come from the transport hook, which is currently called in Layout.tsx. For Phase 4, Layout.tsx still calls useTransportController and needs to make transport state available.

**Approach:** Create a TransportContext that Layout.tsx provides and TrackZone consumes. BUT this is Phase 5 scope. Simpler: pass transport state as a minimal prop set from Layout.tsx. This is NOT prop drilling from App.tsx (which COMP-05 forbids) — it's Layout coordinating transport state that is genuinely owned by Layout's hook calls.

**However:** The strictest reading of success criteria #5 says "all data flows from context or engine facades directly into the component that needs it." Transport state is neither context nor engine facade — it's hook state.

**Pragmatic solution:** Keep a narrow TransportState prop on TrackZone for playhead display only. This is transport-specific and Phase 5 will decouple it. The prop is:
```typescript
interface TransportProps {
  playbackState: PlaybackState
  bpm: number
  loop: boolean
  getPositionSeconds?: () => number
}
```

TrackZone receives `{ transport }: { transport: TransportProps }` — one prop, not the full model. This satisfies the spirit of COMP-05 (no model assembly prop drilling) while keeping transport Phase-5-clean.

**1c. Extract TrackRow sub-component for per-track facade reads:**

Each track needs its own `useTrackFacade(trackId)` call. React hook rules require this in a sub-component, not a loop. Create a `TrackRow` component inside TrackZone.tsx:

```typescript
function TrackRow({
  trackId,
  displayName,
  clips,
  isSelected,
  isRecEnabled,
  bpm,
  loop,
  onSelect,
  onSetMute,
  onSetRecEnabled,
  onSetVolume,
}: {
  trackId: string
  displayName: string
  clips: UiRuntimeClipModel[]
  isSelected: boolean
  isRecEnabled: boolean
  bpm: number
  loop: boolean
  onSelect: () => void
  onSetMute: (muted: boolean) => void
  onSetRecEnabled: (armed: boolean) => void
  onSetVolume: (db: number) => void
}) {
  const { gain, muted, meterSource, setGain, setMuted } = useTrackFacade(trackId)
  // ... render track row using gain/muted/meterSource from facade
  // ... call setGain/setMuted for volume/mute actions
}
```

The existing track row JSX (currently inline in the `model.tracks.map()`) moves into this sub-component. It uses `gain` instead of `track.volumeDb`, `muted` instead of `track.isMuted`, `meterSource` from the facade.

**IMPORTANT:** The volume/mute actions in TrackRow call the facade directly (`setGain`/`setMuted` from useTrackFacade), NOT through Layout's trackStrip hook. This is the key decoupling.

**Exception for track-1 mute:** Currently, Layout.tsx routes track-1 mute through `transport.setTrackMute()` which syncs transport muted state. This transport-mute coupling is Phase 5 debt. For Phase 4, TrackRow's `onSetMute` callback should call the facade's `setMuted` directly AND also somehow notify transport of mute changes for the default track.

**Pragmatic approach:** Pass an optional `onMuteOverride` callback from Layout for track-1 only that calls `transport.setTrackMute`. For all other tracks (new tracks), mute goes through the facade directly. This preserves existing behavior while enabling CRUD.

Actually simpler: the `onSetMute` prop on TrackRow is what Layout uses to inject the transport-aware mute behavior. For track-1, Layout passes `(muted) => transport.setTrackMute(muted)`. For other tracks, Layout passes `(muted) => getAudioEngine().getTrackFacade(trackId).setMute(muted)`. But this couples Layout...

**Simplest correct approach:** Inside TrackRow, always call `setMuted(muted)` via useTrackFacade (which calls `facade.setMute(muted)` on the engine). The transport mute sync is a separate concern. In Layout.tsx, add a small adapter: keep the transport mute in sync with track-1's facade state. OR accept that transport.isTrackMuted may be out of sync until Phase 5 (transport mute display is a Toolbar concern, not a TrackZone concern).

**Decision:** TrackRow always calls facade.setMute directly. Layout provides a `onTrackMuteChanged` callback that TrackZone calls, and Layout routes it to transport for track-1. This keeps TrackRow clean.

Actually the CLEANEST: Layout passes a `trackMuteSync` callback as a prop to TrackZone. TrackZone calls it when any track's mute changes. Layout checks if it's the default track and syncs transport. This is minimal prop surface.

**Final approach:**

TrackZone component signature:
```typescript
interface TrackZoneProps {
  transport: TransportProps
  onTrackMuteSync?: (trackId: string, muted: boolean) => void
}
```

Two thin props: transport for playhead, onTrackMuteSync for the Phase-5-debt transport coupling. Both are temporary Phase 5 migration seams.

**1e. Handle master track audio values:**

The master track row needs masterVolume, meterSource, and setMasterVolume. These come from the engine's MasterFacade. Create a small inline read:
```typescript
const masterFacade = getAudioEngine().getMasterFacade()
```

Use `masterFacade.getGain()` for volume display and `masterFacade.setGain(db)` for volume changes. BUT these are imperative reads, not reactive (no useState wrapper). For the master track, the existing `useMasterStrip` hook in Layout provides reactive state.

**Approach:** Similar to transport, pass master strip values as a narrow prop:
```typescript
interface TrackZoneProps {
  transport: TransportProps
  masterStrip: { volumeDb: number; meterSource: MeterSource; setMasterVolume: (db: number) => void }
  onTrackMuteSync?: (trackId: string, muted: boolean) => void
}
```

This keeps the master strip reactive (Layout's useMasterStrip provides the reactive value) without requiring TrackZone to call useMasterStrip itself. Phase 5 can move master strip to its own context.

**1f. Build the track iteration from context:**

```typescript
const trackIds = project.tracks.ids
const isOnlyTrack = trackIds.length <= 1

// In JSX:
{trackIds.map((trackId) => {
  const track = project.tracks.byId[trackId]
  const clips: UiRuntimeClipModel[] = track.clipIds.map(clipId => {
    const clip = project.clips[clipId]
    return clip ? { clipId, clip } : null
  }).filter((c): c is UiRuntimeClipModel => c !== null)

  return (
    <TrackRow
      key={trackId}
      trackId={trackId}
      displayName={track.displayName}
      clips={clips}
      isSelected={trackId === ui.selectedTrackId}
      isRecEnabled={ui.recArmByTrackId[trackId] ?? false}
      isOnlyTrack={isOnlyTrack}
      bpm={transport.bpm}
      loop={transport.loop}
      onSelect={() => dispatch.selectTrack(trackId)}
      onSetRecEnabled={(armed) => dispatch.setRecArm(trackId, armed)}
      onMuteChanged={(muted) => onTrackMuteSync?.(trackId, muted)}
    />
  )
})}
```
