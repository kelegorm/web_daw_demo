**1c. Resolve device modules for the selected track:**

The selected track's devices need to be resolved to their runtime modules. For track-1, the devices are `['dev-synth', 'dev-panner']` in ProjectDocument. For new tracks, `deviceIds` is `[]` (empty). For master track, `deviceIds` is `['dev-limiter']`.

Device resolution currently goes through `DEVICE_REGISTRY.resolveModule(audioEngine, moduleId)` — but this calls `audioEngine.getSynth(moduleId)`, which only works with the legacyEngineAdapter (deleted in 04-01).

**The device modules (toneSynth, panner, limiter) are React hooks return values**, not simple engine facade reads. `useToneSynth`, `usePanner`, `useLimiter` are React hooks called in Layout.tsx. Their return values (`ToneSynthHook`, `PannerHook`, `LimiterHook`) have reactive state (filter cutoff, bypass state, etc.).

**This means DevicePanel cannot resolve device modules from the engine alone** — the hooks are called in Layout.tsx and their return values are what the device components (SynthDevice, PannerDevice, LimiterDevice) need.

**Approach:** DevicePanel receives the hook return values from Layout.tsx. BUT this is prop drilling, which COMP-05 forbids.

**Alternative:** DevicePanel calls the hooks itself. But the hooks require the graph objects (`_synthGraph`, `_pannerGraph`, `_limiterGraph`) which are module-level in Layout.tsx.

**Pragmatic solution:** Keep a narrow device-modules prop from Layout. This is a Phase 5 concern — device lifecycle management. DevicePanel receives only the device hook instances, not a full model:

```typescript
interface DevicePanelProps {
  deviceModules: Record<string, AnyDeviceModule>
}
```

Layout passes: `{ 'dev-synth': toneSynth, 'dev-panner': panner, 'dev-limiter': limiter }`.

DevicePanel reads selectedTrackId and track info from context, looks up deviceIds from the track, and resolves modules from the prop map. For new tracks (empty deviceIds), DevicePanel shows nothing.

**This is the minimum viable prop surface** — just the reactive hook instances. All track resolution (which track, which devices, display names) comes from context.

Actually, re-reading the success criteria: "all data flows from context or engine facades directly into the component that needs it." Device hook instances are neither context nor engine facades — they're React hook return values. This is a genuine gap that can only be fully solved by moving device lifecycle into the engine or into a device context (future milestone).

**Final decision:** Accept `deviceModules` as a narrow Layout-to-DevicePanel prop. Document as Phase 5 seam. This is architecturally equivalent to the transport prop on TrackZone — a temporary coupling that Phase 5 cleans up.

```typescript
interface DevicePanelProps {
  deviceModules: Record<string, AnyDeviceModule>
}

export default function DevicePanel({ deviceModules }: DevicePanelProps) {
  const project = useProjectState()
  const ui = useUiState()

  // Determine selected track info
  const isMasterSelected = ui.selectedTrackId === project.masterTrack.id
  const selectedTrack = isMasterSelected ? null : project.tracks.byId[ui.selectedTrackId]
  const selectedTrackDisplayName = isMasterSelected
    ? project.masterTrack.displayName
    : (selectedTrack?.displayName ?? '')
  const deviceIds = isMasterSelected
    ? project.masterTrack.deviceIds
    : (selectedTrack?.deviceIds ?? [])

  // Resolve devices: look up from context (Device) + prop (module instance)
  const devices: UiRuntimeDeviceModel[] = deviceIds
    .map(deviceId => {
      const device = project.devices[deviceId]
      if (!device) return null
      const module = deviceModules[deviceId]
      if (!module) return null
      return {
        uiDeviceId: device.id,
        displayName: device.displayName,
        moduleId: device.id,
        moduleKind: device.kind as DeviceModuleKind,
        module,
      }
    })
    .filter((d): d is UiRuntimeDeviceModel => d !== null)

  // Render using existing renderDeviceFromRegistry
  // ...keep existing JSX structure
}
```

**1d. Keep the existing render structure** — the frame, track name strip, device content area, and fade animation all stay. Just source data from context + deviceModules prop instead of the monolithic `model` prop.
